# 97_API_ERRORS

Цель: единый каталог API-ошибок для Web и Mobile, чтобы UI одинаково обрабатывал ответы и показывал предсказуемые сообщения.

## Формат ошибки API
Базовый формат на сервере: `ProblemDetails` (`type`, `title`, `status`, `traceId`, иногда `errors`).

Клиенты (`Web/Mobile`) должны:
- сначала смотреть HTTP status;
- извлекать `traceId` из body;
- маппить status -> `app error code` (таблица ниже);
- показывать пользователю короткое сообщение + код ошибки;
- в логах хранить полный body.

## Каталог кодов (MVP)
| HTTP | App code | Пользовательское сообщение | Действие UI |
|---|---|---|---|
| 400 | `API.BAD_REQUEST` | Некорректный запрос. | Показать баннер ошибки, оставить форму открытой. |
| 400 + `errors` | `API.VALIDATION` | Проверьте корректность введенных данных. | Подсветить форму, не закрывать диалог. |
| 401 | `API.UNAUTHORIZED` | Сессия истекла или недействительна. Выполните вход повторно. | Redirect на `/login` + очистка session. |
| 403 | `API.FORBIDDEN` | Недостаточно прав для выполнения операции. | Redirect на `/forbidden`. |
| 404 | `API.NOT_FOUND` | Запрошенный ресурс не найден. | Показать user-friendly empty/not-found state. |
| 409 | `API.CONFLICT` | Конфликт состояния данных. Обновите страницу и повторите действие. | Reload экрана и повтор действия вручную. |
| 422 | `API.UNPROCESSABLE` | Данные не прошли валидацию. | Показать баннер + оставить форму. |
| 429 | `API.RATE_LIMIT` | Слишком много запросов. Повторите через минуту. | Временный disable кнопки/ретрай с backoff. |
| 500 | `API.INTERNAL` | Внутренняя ошибка сервера. | Показать ошибку + traceId. |
| 502 | `API.BAD_GATEWAY` | Проблема на промежуточном шлюзе. | Показать ошибку, предложить ретрай. |
| 503 | `API.UNAVAILABLE` | Сервис временно недоступен. | Показать ошибку, предложить ретрай. |
| 504 | `API.TIMEOUT` | Сервис не ответил вовремя. | Показать ошибку, предложить ретрай. |
| Other | `API.HTTP_<status>` | Ошибка API (<status>). | Базовый fallback. |

## Реализация в Web
- Форматирование ответа: `src/Chop.Web/Auth/ApiErrorFormatter.cs`
- Нормализация на экранах: `src/Chop.Web/Auth/UiErrorMapper.cs`
- Redirect для auth/RBAC: `src/Chop.Web/Auth/AuthErrorRedirector.cs`

## Требования к Mobile
- Использовать тот же каталог кодов и те же пользовательские сообщения.
- Redirect-логика заменяется на mobile-эквивалент (`logout -> login screen`, `forbidden screen`).

## Правила расширения
- Любой новый app-code добавляется в этот файл и в реализацию форматтера.
- Нельзя показывать пользователю raw stack trace.
- `traceId` обязателен в тексте диагностики для 5xx.
