# UI — Mobile (MAUI Blazor Hybrid)

## 1. Общие правила
- Один апп, разные разделы по роли (CLIENT или GUARD).
- Важные состояния сети: ONLINE / OFFLINE / SENDING / SENT(ACK) / ERROR.
- Все сетевые ошибки — в понятном виде + повторить.

## 2. Клиент (CLIENT)
### Экраны
1) /client/sos
- большая кнопка SOS
- защита от случайного нажатия: “удерживать 2 секунды” (MVP)
- состояния:
  - Получаем геолокацию…
  - Отправляем…
  - Доставлено (ACK) + время
  - Нет сети: “поставлено в очередь, отправим при появлении интернета”
2) /client/profile
- ФИО, телефон
- адреса (список)
- период обслуживания и статус оплаты (read-only)
3) /client/contacts
- телефоны/адреса ЧОП, кнопка “позвонить”
4) /auth/forgot (если реализуем)
- восстановление/сброс

## 3. Охранник (GUARD)
### Экраны
1) /guard/shift
- кнопка “Заступил”
- выбор поста/экипажа (если не назначено)
- статус смены + время начала
2) /guard/incidents
- список назначенных активных заявок
- открыть карточку
3) /guard/incidents/{id}
- данные: адрес, координаты, телефоны клиента (если разрешено)
- кнопки статусов: Принять → В пути → На месте → Завершить
- комментарий обязателен при завершении
4) /guard/settings
- позывной/рация (read-only или ограничено)

## 4. Состояния геолокации
- Разрешение не выдано → объяснение и кнопка “Открыть настройки”.
- Гео недоступно → отправить SOS без координат (если PRD позволяет) + отметить “no-location”.
- Режимы:
  - NORMAL: пинг 10 минут
  - ACTIVE_INCIDENT: пинг 10 секунд (только пока есть активная заявка)

## 5. Минимальная аналитика (внутренняя)
- Последняя отправка пинга (время).
- Последний ACK по SOS.
