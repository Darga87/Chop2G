# 100_DATA_RETENTION

Цель: базовая политика хранения и удаления данных (MVP), чтобы снизить риск бесконтрольного роста PII/технических логов и поддержать audit/compliance.

## Принципы
- Храним только необходимые данные.
- Для PII применяем минимально достаточный срок.
- Удаление/архивация выполняются фоновой задачей (batch + retry + audit).
- Любое отклонение от политики фиксируется решением в `docs/90_DECISIONS.md`.

## Политика по сущностям (MVP)
| Сущность / таблица | Срок хранения | Действие по истечении | Основание |
|---|---|---|---|
| `audit_log` | 365 дней | Архивация/удаление старых записей | Расследования, безопасность |
| `outbox_messages` (published/failed) | 30 дней | Удаление | Техническая телеметрия |
| `notification_deliveries` | 90 дней | Удаление | Диагностика доставки |
| `notification_outbox` (sent/failed) | 90 дней | Удаление | Диагностика оповещений |
| `alert_events` (`RESOLVED`) | 180 дней | Архивация/удаление | Операционный анализ |
| `guard_locations` | 30 дней | Удаление старой истории | Гео-данные повышенной чувствительности |
| `incident_status_history` | 365 дней | Архивация | Трассировка инцидентов |
| `incidents` | 3 года | Архивация (без удаления в MVP) | История событий, претензионка |
| `refresh_tokens` (revoked/expired) | 30 дней | Удаление | Security hygiene |
| `invitations` (used/expired) | 30 дней | Удаление | Security hygiene |
| `password_resets` (used/expired) | 30 дней | Удаление | Security hygiene |
| `payments` / `bank_import*` | 5 лет | Архивация | Финансовый учет |

## Техническая реализация (рекомендация)
- Daily background job:
1. выбирает кандидатов по `createdAt/updatedAt/status`;
2. удаляет/архивирует ограниченными батчами;
3. пишет summary в `audit_log` (`retention.cleanup.*`).

## Ограничения MVP
- Юридические/регуляторные сроки (по стране/договору) требуют отдельной валидации перед production.
- Для финансовых данных срок 5 лет считать консервативным baseline до подтверждения бухгалтерией/юристом.
